# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this Zed extension, please report it privately.

Currently, as this is a language extension with minimal attack surface, security issues are expected to be rare. However, we take all reports seriously.

## Supported Versions

We will provide security updates for the latest release version of this extension.

## Reporting Process

1. **Do not** create a public GitHub issue for security vulnerabilities
2. Contact the maintainers directly through GitHub private messages or email
3. Provide a detailed description of the vulnerability and steps to reproduce
4. Allow reasonable time for investigation and patching

Thank you for helping keep this project secure.
